## 풀이

### 분석

문제를 분석한 내용을 작성합니다.

### 취약점
-프로그램은 "인덱스는 0~9 사이"라고 안내하지만, 실제 코드에서는 입력된 인덱스가 유효한 범위인지 전혀 검증하지 않습니다. 따라서 배열 범위를 벗어난 인덱스를 입력해도 그냥 해당 메모리 위치에 값을 써버리는 OOB Write 취약점이 존재합니다.

### 익스플로잇
-함수가 실행될 때 스택에는 함수가 끝난 뒤 어디로 돌아갈지를 나타내는 return address가 저장됩니다. objdump로 main 함수를 분석하면 배열의 시작 위치는 rbp-0x30이고, return address는 rbp+0x08에 위치합니다. 두 위치의 거리는 0x38 즉 56바이트이고, int가 4바이트이므로 56 / 4 = 14가 되어 index 14가 정확히 return address를 가리킵니다. 그런데 64비트 시스템에서 주소는 8바이트이지만 scanf("%d")는 4바이트씩만 씁니다. 따라서 하위 4바이트(index 14)에 print_flag 주소인 0x4011c9를 쓰고, 상위 4바이트(index 15)에는 0을 써서 주소가 올바르게 완성되도록 해야 합니다.


# 풀이 코드 예시
우선 ctf에서 주어진 nc 명령어를 입력합니다. 그런후에 
2          ← 총 2번 수정
14         ← return address 하위 4바이트
4198857    ← print_flag 주소 (0x4011c9 → 10진수 변환)
15         ← return address 상위 4바이트
0          ← 0x00000000
이렇게 입력했습니다.
