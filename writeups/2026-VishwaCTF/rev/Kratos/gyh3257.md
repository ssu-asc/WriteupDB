# Kratos

## 문제 설명
> Unity IL2CPP로 빌드된 Android APK가 주어진다. 올바른 입력을 제출하면 flag를 출력한다.

- 파일: `Kratos.apk`

## 풀이

### 분석

APK 내부 구조를 확인한다.

```bash
unzip -l Kratos.apk | grep -E "\.so|\.dex|lib/"
```

`libil2cpp.so`, `libgame.so`, `classes.dex` 등이 존재하며 Unity IL2CPP 빌드임을 확인했다. `global-metadata.dat`도 함께 추출한다.

### C# 로직 복원

cpp2il을 사용해 IL2CPP 바이너리에서 C# 코드를 복원한다.

```bash
dotnet Cpp2IL.dll \
  --game-path Kratos.apk \
  --output-as diffable-cs \
  --output-to il2cpp_output/
```

`Assembly-CSharp/buttonBehaviour.cs`에서 핵심 로직을 발견했다.

```csharp
public class buttonBehaviour : MonoBehaviour
{
    private string targetHash;
    private byte[] flagCipher;
    private byte xorKey;

    private string ComputeSHA256(string rawData) { }
    private string DecryptFlag() { }
    public void OnSubmitClicked() { }
}
```

입력값을 SHA256으로 해시한 뒤 `targetHash`와 비교하고, 일치하면 `flagCipher`를 `xorKey`로 XOR 복호화해 flag를 출력하는 구조다.

### 데이터 추출

`_PrivateImplementationDetails_.cs`에서 정적 배열 데이터를 확인했다.

```
__StaticArrayInitTypeSize=29 (flagCipher):
3C 03 19 02 1D 0B 29 3E 2C 11 13 5A 1F 35 0D 5A
5D 35 01 18 5E 1E 5A 19 35 2D 5A 2E 17
```

### XOR 브루트포스

xorKey는 단일 바이트(0x00~0xFF)이므로 256가지를 전부 시도한다.

```python
cipher = bytes([0x3C, 0x03, 0x19, 0x02, 0x1D, 0x0B, 0x29, 0x3E,
                0x2C, 0x11, 0x13, 0x5A, 0x1F, 0x35, 0x0D, 0x5A,
                0x5D, 0x35, 0x01, 0x18, 0x5E, 0x1E, 0x5A, 0x19,
                0x35, 0x2D, 0x5A, 0x2E, 0x17])

for key in range(256):
    result = bytes([b ^ key for b in cipher])
    try:
        text = result.decode('ascii')
        if all(32 <= c < 127 for c in result):
            print(f"key=0x{key:02X}: {text}")
    except:
        pass
```

`key=0x6A`일 때 의미 있는 문자열이 출력됐다.

```
key=0x6A: VishwaCTF{y0u_g07_kr4t0s_G0D}
```

## 배운 점

- Unity IL2CPP APK는 일반 ELF 분석 도구로 바로 분석이 어렵고, cpp2il 같은 전용 툴로 C# 로직을 먼저 복원해야 한다.
- `_PrivateImplementationDetails_`에 정적 배열 데이터가 hex blob으로 저장되므로 여기서 암호화된 데이터를 추출할 수 있다.
- xorKey가 단일 바이트면 브루트포스로 충분히 복호화 가능하다.
