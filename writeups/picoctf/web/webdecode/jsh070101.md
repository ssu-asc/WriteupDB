# 문제명

## 문제 설명

>Author: Nana Ama Atombo-Sackey

Description
Do you know how to use the web inspector?
Start searching here to find the flagi

-http://titan.picoctf.net:65010/

## 풀이

. 분석 과정
단계 1: 웹사이트 탐색
링크에 접속하면 home, about, contact 세가지 목록을 가진 사이트가 시작됨. 따라서 asc 에서 웹 문제 풀었던대로 개발자 창 켜고 이것저것 뒤져보기 시작함. 
그날 기초적으로 배운 내용은 base64 즉 일종의 인코딩 내용과 토큰을 가로채서 변경시키는 과정을 상기하며 살핌.

단계 2: 탐색 및 코드 검사
웹사이트 상단 목록들을 이리저리 둘러보며 개발자 모드로 이상한 부분들을 찾아봄. 혹시나 로그인이 있나 살펴보기도 하고, 소스나 네트워크 창도 살펴봄. 근데 정말 아무것도 없는 느낌이라
상당히 당황스러웠음. 그래서 혹시나 완전 쉬운 문제인가 하고 저번에 문제 풀며 배운 서치 기능으로 flag와 base64에서 쓰이는 == 표시도 검색해봤지만 영 성과가 없었다.

단계 3: 숨겨진 문자열발견
개발자 도구의 elements 에서 계속 이것저것 눌러보다가 다른 문자열과는 다르게 유독 긴 문자열을 about 창에서 발견함. 

수상한 값: cGljb0NURnt3ZWJfc3VjYzNzc2Z1bGx5X2QzYzBkZWRfMWY4MzI2MTV9 이런식으로 나와있었는데 == 표시가 안떠 먼가 싶어 메모장에 적어만 두고 한 10분은 더 탐색했다.

3. 해결 방법 

결국엔 그 온라인으로 숨겨진 코드를 찾아낼 수 있는 cyberchef 사이트에서 위의 값을 base64 인코딩을 하자 숨겨진 플래그가 등장했다. 

Recipe: From Base64 적용

Input: cGljb0NURnt3ZWJfc3VjYzNzc2Z1bGx5X2QzYzBkZWRfMWY4MzI2MTV9 입력

Output: picoCTF{web_succ3ssfully_d3c0ded_1f832615}.

4. 최종 결과
picoCTF{web_succ3ssfully_d3c0ded_1f832615}

느낀점 : 나중에 찾아보니 웹 검사기(web inspector)는 일종의 개발자 도구였고, 쉬운수준이긴 했지만 처음으로 인공지능 없이 문제를 살펴보며 플래그의 단서를 파악한 ctf 문제여서 굉장히 흥미를 느꼈다. 
하지만 비교적 문제가 맥없이 풀려 아쉬운 점이 있었다.
